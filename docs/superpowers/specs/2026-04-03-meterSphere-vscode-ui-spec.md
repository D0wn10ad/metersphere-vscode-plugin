# UI/UX Spec — Phase 1 and Phase 2 (Draft)

- Phase 1: WebView-based API debugger with status indicators and commands for token/workspace/project set; No Navigator yet.
- Phase 2: Introduce Core Navigator (TreeView) with Phase C command-driven UX and on-demand sync integration; WebView remains primary editing surface.
- Keyboard and discoverability: Commands visible in Command Palette; optional shortcuts where relevant.
- Flows: Token/Workspace/Project update flows; error handling prompts; syncing prompts for conflicts.
